# Гармония - исходники сайта

Это статическая версия сайта проекта "Гармония" для передачи разработчикам.

## Состав

- `index.html` - структура страниц и контент.
- `styles.css` - все стили, адаптив, анимации и визуальные состояния.
- `script.js` - меню, модальные окна, слайдеры, переключение инфраструктуры, курсор, скроллспай, лайтбокс.
- `assets/` - локальные изображения, логотипы, шрифты и графика сайта.

## Как открыть

Можно открыть `index.html` напрямую в браузере.

Для корректной проверки как на сервере:

```bash
python -m http.server 8088
```

После запуска открыть:

```text
http://127.0.0.1:8088/
```

Ролик приглушён (`muted`, обязательно для автовоспроизведения) и зациклен (`loop`), автоматически запускается, когда раздел появляется во вьюпорте (порог 50% в `IntersectionObserver`), и ставится на паузу при прокрутке в любую сторону от него.

## Правки производительности

Нашлась и устранена причина жалоб на медленную загрузку и подвисания:

- **Главная находка:** `preload="metadata"` на `<video>` не гарантирует частичную загрузку - в этом браузере (и не только в нём) при наличии `<source>` в разметке видео всё равно запрашивалось целиком (10 МБ) сразу при открытии страницы, независимо от прокрутки. Источник видео теперь не прописан в HTML (`data-src` вместо `src`), а добавляется скриптом только когда раздел "Видео о проекте" приближается к зоне видимости (`IntersectionObserver` с `rootMargin: 600px`). Вес первой загрузки страницы упал с ~12 МБ до ~1.6 МБ.
- Три независимых обработчика `scroll` (хедер, полоса прогресса, scrollspy), каждый со своим синхронным чтением `getBoundingClientRect`/`scrollHeight`, объединены в один тик через `requestAnimationFrame` - это исключает повторные пересчёты layout на каждое сырое событие скролла.
- Магнитные кнопки, courser-подсветка карточек и параллакс лого в hero больше не вызывают `getBoundingClientRect()` на каждое движение мыши - размеры элемента считываются один раз по `mouseenter` и кэшируются.
- Фоновые анимации (`heroDrift`, `fullSceneFade`, `locationScenePan`, бегущая строка) теперь стоят на паузе, пока их раздел не станет видимым (`.panel.is-current`, на основе уже существующего `IntersectionObserver`) - раньше все они крутились одновременно на всей странице независимо от прокрутки.
- У `.grain` убран `mix-blend-mode: overlay` (дорогая операция смешивания на весь вьюпорт на каждый кадр); зерно теперь рисуется обычной полупрозрачностью - визуально почти неотличимо на тёмной палитре, но заметно дешевле.
- Уменьшен радиус блюра у `backdrop-filter` хедера (22px -> 14px, единственный always-on блюр на странице) и радиус размытия у `--shadow` (120px -> 64px), используемого во многих карточках и рамках изображений.

## SEO

- `<title>` и `meta description` расширены под реальные УТП (300-470 м2, 15 минут до моря, старт продаж 10 июля), добавлены `meta keywords`, `meta robots`, `meta author`.
- Open Graph дополнен: `og:locale`, `og:site_name`, `og:url`, размеры и alt для `og:image`. Добавлены полноценные `twitter:title/description/image`.
- Добавлена микроразметка JSON-LD (`schema.org/ApartmentComplex`) с адресом, количеством юнитов и ключевыми удобствами - помогает поисковикам понять, что это за объект, и повышает шанс на расширенные сниппеты.
- Добавлены `robots.txt` и `sitemap.xml`.
- **Важно:** канонический домен `https://harmony-yalta.ru/` - это временная заглушка (реальный домен ещё не определён). Перед публикацией нужно заменить его на настоящий адрес сайта в: `index.html` (`canonical`, `og:url`, `og:image`, `twitter:image`, JSON-LD `url`/`image`), `robots.txt` (`Sitemap:`) и `sitemap.xml` (`loc`).
- Проверено: ровно один `<h1>` на странице (скрытый, для скринридеров/SEO), 8 `<h2>` по разделам, у всех `<img>` есть `alt`.
